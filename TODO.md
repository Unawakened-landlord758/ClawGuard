### 当前判断

- 不是做新的 OpenClaw，而是做 OpenClaw 生态位里的“安全控制层 / 安全卫士”
- 对外统一品牌：`ClawGuard`；中文统一品牌：**龙虾卫士**
- 对外主 slogan：**The antivirus for OpenClaw**
- 最强公众钩子不是“更强能力”，而是“让 AI 能干活，但不能乱来”
- 最值得优先击中的焦点：转账/红包、敏感信息外发、危险命令、恶意 skills、看不懂 AI 在干什么
- 需要把零散安全能力组织成“分层需求 + 多级防护 + 多引擎检测 + 多级处置 + 用户自定义策略”的方法体系
- 首发产品最适合做：危险动作审批 + 风险体检评分 + 基础 skills 验毒 + 基础泄密阻断
- OpenClaw 首宿主不应再默认走 Wrapper/Launcher；正确首发形态是原生插件 + 插件自带审批/审计/设置页面
- 审批闭环第一版不能假装有原生 resume；应明确采用 `PendingAction + AllowOnceGrant + 重试放行`
- 首发产品不应以粗暴错误结束风险动作，而应返回语义化提醒，引导用户去审批页或确认链路处理
- 当前代码已真实验证：`exec` 主链支持 pre + post 审计闭环；outbound 已同时覆盖工具级 `message` / `sessions_send` 审批链与宿主级 `message_sending` 直接发送 hard-block；`workspace mutation` 已有插件级最小 pre + post 闭环，覆盖 `write` / `edit` / `apply_patch` 的共享 Core 判定、pending/block 入口与结果审计
- 当前插件最适合作为 OpenClaw 首宿主验证基线；下一步应继续减少 plugin runtime 与 Core 的重复，而不是回头做 Wrapper
- 当前首个 install demo 已具备最小元数据与文档基线：推荐本地路径安装、可选本地 tarball、包名仅作占位元数据 / 未来兼容命名，仍不应宣称已发布
- 当前 install demo 只应讲“本地安装 + 页面 smoke + fake-only 演示记录”，不应包装成正式 release，也不应暗示 npm registry 已存在
- 当前仓库已恢复本地验证基线：`pnpm typecheck` 与 `pnpm test` 均可通过；tarball surface check 也已覆盖本地打包文件面
- 当前 install-demo 已按 OpenClaw 官方插件打包建议切到构建产物入口：`openclaw.extensions` 指向 `dist/index.js`，本地 `openclaw plugins install .\plugins\openclaw-clawguard` 已能成功加载插件
- 当前 OpenClaw 插件 API 已足够承载插件自有页面（`/plugins/clawguard/*`），但 Control UI 左侧导航仍是 Core UI 硬编码，暂无正式插件注册 `Security` 标签页的 API
- 当前 `workspace mutation` 已开始携带共享 Core 的 `workspace_context.operation_type`：`edit` 已收紧到高置信 identifier-family rename-like，`apply_patch` 已支持 Add / Delete / Modify / RenameLike，path-pair 场景也已能在高置信时补充 `rename-like`，低置信则保守回退 `modify`
- 当前 release 文档工作已完成 first usable version 汇总草稿：external / internal release notes、announcement 与 acceptance checklist 均已落盘，可在真正对外发布前继续润色
- 当前 **first usable version** 口径已固定为：本地路径优先安装 + 以 `/clawguard*` 为浏览器入口的五页 smoke（`/clawguard` → `/clawguard/checkup` → `/clawguard/approvals` → `/clawguard/audit` → `/clawguard/settings`）+ fake-only 的 `exec` / minimal outbound / `write` / `edit` / `apply_patch` workspace mutation 演示
- 当前 **first usable version** 明确 **不包括**：registry publish、GA、真实危险执行、真实红包/转账、真实 outbound 验证、完整 outbound/workspace 覆盖、Control UI 左侧 `Security` 导航或 patched UI 集成
