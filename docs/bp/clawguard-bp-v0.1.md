## 第 2 页：为什么这个问题现在必须被解决

### 标题

**OpenClaw 越火，安全控制问题越不能再留给用户自己扛**

### 文案

OpenClaw 已经不再只是极客工具，而是在快速走向普通用户与小团队。

但用户面对的是一种新的风险结构：

- AI 可能被一句话诱导去执行危险动作
- 用户不知道它下一步要做什么
- 关键动作发生前没有清晰的刹车机制
- 事后也缺少可追溯、可解释的风险记录

对大多数用户来说，他们担心的不是 AI 不够聪明，而是：

> **它太听话、权限太大，而自己看不清、拦不住、追不回。**

### 页面建议元素

- 左边：OpenClaw adoption / 热度上升
- 右边：风险焦虑上升
- 底部：一句高亮金句

## 第 3 页：我们看到的真实用户需求

### 标题

**普通用户真正需要的，不是安全术语，而是安全感、控制感和解释权**

### 文案

我们判断，OpenClaw 时代会出现一类与 PC 时代安全软件类似的用户需求：

- 先告诉我现在是不是有风险
- 再告诉我具体风险在哪
- 关键动作发生前，给我最后确认权
- 事后能够追溯发生了什么

这类需求并不要求用户理解底层安全原理，但要求产品把复杂风险翻译成普通人可理解、可操作的体验。

### 可视化建议

用“PC 时代安全卫士”到“AI Agent 时代安全卫士”的类比图：

- 过去：木马 / 病毒 / 恶意软件
- 现在：提示注入 / 风险动作 / skills 风险 / 泄密外发

### 口播备注

我们做的更像 360 安全卫士之于大众用户，而不是只给专业安全团队使用的重型平台。

## 第 4 页：ClawGuard 是什么

### 标题

**ClawGuard = OpenClaw 的安全控制层**

### 文案

ClawGuard 面向 OpenClaw 生态，提供一套“先解释、再审批、能阻断、可审计”的安全控制能力。

当前核心能力包括：

1. **危险动作审批**
   - 删除文件、外发消息、执行命令等关键动作先过人
2. **风险体检与安全评分**
   - 让用户知道当前实例的总体安全状态
3. **skills 验毒**
   - 对扩展与能力包做安装前风险判断
4. **基础泄密阻断**
   - 阻止敏感信息被外发
5. **审计与回放**
   - 清楚记录什么时候发生了什么、为什么危险、谁做了决定

### 页面建议元素

五张能力卡片或五个图标横排。

## 第 5 页：最强产品故事

### 标题

**一句群消息，差点让 OpenClaw 发红包；ClawGuard 把最后一票还给了人**

### 文案

这是我们最核心的 north star demo：

1. 群聊里出现一条诱导消息
2. OpenClaw 准备执行资金类高风险动作
3. ClawGuard 在执行前识别风险并拦截
4. 系统给出可理解的风险解释
5. 动作进入审批流，最终由用户决定是否放行
6. 全过程留痕，事后可追溯、可复盘

### 高亮句

> 我们要证明的不是“AI 完全不犯错”，而是“关键动作不能黑盒发生”。

## 第 6 页：为什么我们现在有机会做这件事

### 标题

**竞品很多，但“OpenClaw 的安全卫士”这件事还没有被真正产品化**

### 文案

当前市场上已经出现多种相关方案：

- 更安全的 OpenClaw 替代版
- 面向开发者的 guardrails / policy 工具
- 偏扫描和平台化的安全工具

但它们多数存在三个问题：

1. 更像工程设施，而不是用户产品
2. 更偏开发者和企业，而不是普通用户与轻团队
3. 更强调检测能力，而不强调解释权和控制权

ClawGuard 的差异化切口是：

- 依附 OpenClaw 生态，而不是替代 OpenClaw
- 面向普通用户与小团队
- 用“审批 + 阻断 + 解释 + 审计”建立安全心智
- 以开源方式积累规则资产与社区共建能力
